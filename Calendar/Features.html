Features

Database

Remote MongoDB backend.
Different data types split into “collections”.
Stores user login information (cryptographically secure) and tasks/schedules.
Access tokens provided by user login.
Built-in search, sort, and ordering functions assist schedule generation.

Task Management

Task management allows for the inputting, deleting, and editing of tasks into a schedule.
Allows tasks to be prioritized depending on the due date.
Multiple types of task can be inputted, each class represents a different time range for the task.

Schedule Management

Allows the creation of schedules that are set from a certain time range and allows the generation of “best time” schedules.
“Best time” Schedules must use an existing schedule to be generated.
“Best time” Schedules are generated from tasks inputted and based off of priority.
After the creation of the "Best Time" schedules, they display tasks inputted in a certain time frame.

Storage

Consists of two components: Schedule storage and task storage.
These components are responsible for holding the tasks or schedules associated with each user account.
Connection with the database allows transfer of data from remote to local storage.
Allows for the saving of data locally so that the user will be able to access it later.
